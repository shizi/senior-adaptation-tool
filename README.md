# JoyCode Zero 设计稿适老化适配技能

服务于内部设计师的 Zero 设计稿通用适老化适配工具。一键将任意 Zero 设计稿适配为**大字版（1.15x）**或**长辈版（1.3x）**。

## 工作链路（V2.6）

```
在 Zero 中定位原画板（鼠标选中 / 节点 ID / 名称） → Zero MCP 读节点结构 → 按规则生成适配版本 → Zero MCP 写回原画板右侧

（可选）若提供 Deco 地址 → Deco MCP 拉取源码作为辅助参考
```

- ✅ **V2.2 三画板架构**：原画板 → 预处理画板（保留，作调试参考） → 大字版 + 长辈版
- ✅ **预处理画板**（V2.2）：INSTANCE 自底向上 detach（含出错回滚兜底）+ 行高强制 120% 覆盖
- ✅ **用户指定不放大节点**（V2.2 长辈版）：实际开发用固定切图实现的区域，可由用户标注后整体保持原尺寸
- ✅ **V2.2 删除间距同步**：实测 padding ×1.3 是错的，文字撑大容器后间距自然合理
- ✅ 字号四舍五入取整、最小 12px
-✅ 字号上限阈值（V2.1）：≥24pt（大字版）/ ≥26pt（长辈版）保持原值
- ✅ textAutoResize 智能选择（V2.1）：HEIGHT 保留、NONE 改 HEIGHT
- ✅ 自动 HUG 高度延展、画板兜底 resize
- ✅ 智能 INSTANCE 处理（默认尝试 detach，出错回滚降级到 OFF 模式）
- ✅ 特殊字号保护（¥ + 价格视觉协调）
- ✅ ICON 默认不放大（V2.5.1 起，按需由设计师手动微调）
- ✅ 弹窗 / 浮层处理（外框宽度严格保持，仅纵向 HUG）

## 使用

在 JoyCode IDE 对话中触发关键词：**"适老化"** / **"大字版"** / **"长辈版"**

**推荐**：在 Zero 中先**鼠标选中**要适配的原画板，然后告诉我"做大字版/长辈版"即可。

**或者**：告诉我目标画板的节点 ID（如 `48:2670`）或名称。

**可选**：补充 Deco 链接作为辅助参考（不必须）：

```
帮我做这个设计稿的大字版和长辈版：
//UI2Code?generateId=2057303085558005762&inspectRatio=1&rect=0_0_375_812&styleUnit=px&type=react-taro
```

## 交互流程（v2.6：严格逐步式）

skill 在收到「适老化/大字版/长辈版」关键词后，**按下面顺序逐步推进，每一步只做该步的事**，不会把多个问题一次性丢给用户：

1. **自检 Zero Design MCP 是否连通**（只读探测 `get_design_metadata`）；未连通就一句话提示用户去连，等待回复
2. **只问一句：要做哪个版本？**（大字版 / 长辈版 / 两个都要）
3. **只问一句：稿子在哪？**（仅当用户既没选中又没给链接时才问）
4. **自己读稿 + 审 Auto Layout 覆盖率**，把 `FRAME 总数 / 已开 Auto Layout 数 / 覆盖率 / 特殊组件候选 / 风险提示` 汇报给用户
5. **基于审稿结果，最多再确认一次**：候选特殊组件是否照单全收；Auto Layout 覆盖率偏低时是否仍坚持继续
6. 用户确认 → 复制画板 → 改属性 → 截图回传验证

### Auto Layout 覆盖率门槛

| 覆盖率 | 处置 |
|--------|------|
| ≥ 70%  | 健康，直接推进 |
| < 70%  | 严肃强调，两选一：① 先去 Zero 手动补 AutoLayout 再来；② 维持现状直接跑（skill 会对绝对定位 Frame 做 x/y 双向位移让位，最终画板裁切回 375×812） |

> 历史 v2.6 曾使用 80% / 50% 三段阈值，已在 v2.7.1 起统一为单阈值 70%，与 SKILL.md §0.4 一致。

### 严禁的开场（违反即视为流程错误）

- ❌ 一上来就把"版本 + 特殊组件 + 设计稿链接"一次性问完
- ❌ 还没确认 Zero MCP 是否连通就直接问业务问题
- ❌ 还没读稿就让用户凭空列举特殊组件
- ❌ 还没审核 Auto Layout 覆盖率就开门见山改字号

## 前置依赖

> **运行环境说明**：本 skill 既可在 JoyCode IDE 中使用，也可被打包直接嵌入 Zero 使用。
> - **IDE 场景**：Zero Design MCP 是**硬门禁**（必须手动连通），用于读写 Zero 设计稿
> - **Zero 内置场景**：宿主自带 Zero Design 能力，MCP 是**可选项 / 由宿主自动满足**，无需用户额外配置
>
> skill 的 Step 0 自检失败时，提示语会区分这两种场景，分别引导。

- **Zero MCP（核心能力）**：用于读取/写回 Zero 设计稿
  - IDE 场景：用户需在 MCP 设置里连上 `zero-design`
  - Zero 内置场景：宿主默认提供，无需用户操作
- **Deco MCP**：**可选**，提供时用于补充语义 / 源码信息

## 已沉淀的实战教训

- **HUG 设置**：仅作用于 Auto Layout FRAME，严禁对 GROUP / 非 Auto Layout FRAME 设 HUG（GROUP 自动包络会被污染）
- **长辈版特殊处理**：顶部头部 banner 不要作整体放大（1.3x 会超出 375 宽度被裁）
- **标签父容器联动**：严格遵守约束 A/B，仅紧贴 wrapper 才联动（否则画板高度会从 ~830 膨胀到 2000+）
- **INSTANCE 默认不解绑**：直接 rescale 即可；仅当字号不生效时启用 detach + 出错回滚
- **AutoLayout 预绑定默认 OFF**：保守路径（绝对定位 + 高度兜底）更稳定
- **textAutoResize 智能选择（V2.1）**：原 HEIGHT 保留，强行改 WIDTH_AND_HEIGHT 会让同行多文本互相覆盖（典型如商品标题挤穿价格）
- **字号上限阈值（V2.1）**：装饰大标题（22pt+）继续放大会破坏顶栏 / 标题视觉版式
- **V2.2 删除间距同步**：实测 padding ×1.3 是错的，文字字号撑大容器后间距自然合理；真正呼吸感由"标签/图文徽章/特殊处理组件整体 rescale"承担
- **V2.2 三画板架构**：原画板 → 预处理画板（保留作调试参考）→ 大字版 + 长辈版，两版本共享预处理 detach 与行高 120% 成果
- **V2.2 detach 出错回滚**：detach 后扫描根 Frame 子节点 y 离群即丢弃 clone 重做（不 detach），实测频繁触发，是稳定性的关键防御
- **V2.2 用户指定不放大节点**：实际开发用固定切图实现的区域，可由用户标注后整体保持原尺寸（判断标准：代码里以 `<image>` 引用且尺寸固定的资源）
- **V2.5.2 门禁简化**：Zero MCP（已连接 + 能定位画板）成为 IDE 场景的硬门禁，Deco 降级为可选辅助；这样设计师在 Zero 选中画板就能直接跑，不必每次都查 generateId（v2.6 进一步：Zero 内置场景该门禁由宿主自动满足）
- **V2.5.3 排除区域全程不动**：用户标注"不放大"节点不仅 1.15/1.3 阶段跳过，连预处理画板的"行高 ×1.2 + 段距清零"也跳过——否则 banner 切图区的文字会吃到 1.0→1.2 的隐性行高变化，与原稿不一致
- **V2.5.3 祖先距离判断**：当用户标注的"放大区"和"不放大区"在节点树上嵌套时（如"评选标准" 在"背景氛围"内），按"白名单祖先距离 vs 黑名单祖先距离，谁离 TEXT 近谁生效"判定；避免外层规则把内层精细标注覆盖掉
- **V2.6 逐步式开场**：开场不再一锅端问"版本+特殊组件+链接"。流程改为：先自检 Zero MCP → 只问版本 → （需要时）只问稿源 → 自己读稿审 Auto Layout 覆盖率 → 把候选特殊组件 + 风险点汇报给用户再确认。**Why**：之前一次性丢 3 个问题用户反感，且特殊组件由用户凭空列举容易漏；让 skill 先扫稿再补充确认更高效
- **V2.6 Auto Layout 覆盖率门禁**：改稿前必须算一次覆盖率。≥80% 直接推进；50%~80% 提示需人工复核；<50% 高风险，先告知再让用户拍板。**Why**：字号放大全靠容器自动撑高，覆盖率低的稿子放大后版式必乱，提前暴露风险比改完返工成本低
