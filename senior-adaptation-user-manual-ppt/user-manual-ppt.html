Workflow · 六步使用流程
03 / 07
从选中画板开始

一次完整协作，只有六步。

Designer × AI Pipeline
01
选画板
在 Zero 中鼠标选中，或提供节点 ID / 名称。
02
定版本
大字版、长辈版，或两个都要。
03
看覆盖率
≥70% 继续；低于 70% 先决定是否补 Auto Layout。
04
标不放大
图片圈选、节点名、坐标范围都可以。
05
写回 Zero
生成预处理、大字版、长辈版新画板。
06
人工验收
看裁切、挤穿、固定切图、低对比度。
演示时按方向键可逐步点亮流程

Annotation · 图片标注
04 / 07
不要把标注做复杂

圈出来就够了。

标注不是为了让 AI 看清每个像素，而是告诉它哪些区域在开发里是固定切图、精调导航或不该被文字放大影响的视觉资产。

图片标注可以直接发给 AI。
不需要把“不放大”的地方单独放大截图。
Rule · 标区域边界，不标像素细节
图片
整张画板截图 + 红框、箭头、备注。
节点名
例如“顶部导航”“京东真榜 logo 区”。
坐标
例如 y=0 到 y=88 的顶部区域。
混合
截图圈选 + 文案说明通常最稳。
图片是协作输入，不是视觉交付
04 / 07
No Scale · 典型排除区
05 / 07
自动跳过系统栏

这些地方
别放大。

StatusBar 和 Home Bar 会自动跳过。你要主动标注的，是那些“设计稿里看起来可编辑，开发里实际固定”的区域。

顶部导航
返回、标题、右侧按钮横向空间紧，易挤出 375。
Tab 区域
底部 tab、频道 tab、一级 tab 容易折行或挤穿。
Banner 头图
营销视觉和品牌大字通常是固定切图或精调资产。
徽章组件
TOP 排名、金银铜标、品牌 logo 区优先保持原状。
不放大 = 文字、图片、所有子节点都不动
05 / 07
Output · Review
06 / 07
跑完以后看什么

AI 生成新画板，
设计师决定它能不能上线。

生成结果

原画板 → 预处理 → 大字版 → 长辈版

skill 会统一行高、按规则放大文字，并把新画板裁切回 375×812，方便并排对照。它不会改原稿，也不会自动补 Auto Layout。

验收重点

裁切、挤穿、误放大、低对比度

尤其看多行标题、价格符号、导航 tab、固定切图、右侧/底部裁切提示。覆盖率低于 70% 的稿件，要重点查重叠和错位。

自动化负责初稿，设计师负责判断

Takeaway · 核心观点
07 / 07
最后记住一句
适老化适配， 本质是一次 结构化设计协作。

你不用把每个细节都交给自动化。你只需要把边界说清楚：哪里能变，哪里不能变，以及变完如何验收。
